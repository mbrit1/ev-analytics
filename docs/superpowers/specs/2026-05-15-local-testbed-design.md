# Spec: Local Testbed with MSW

Status: Proposed
Date: 2026-05-15
Topic: Local Development & UI Testing Environment

## 1. Purpose
To provide a high-fidelity, safe, and isolated local development environment that allows for testing the complete "Offline-First" lifecycle (Auth, Data Entry, Sync) without requiring a real Supabase connection or modifying production data.

## 2. Success Criteria
- **Bypass Login:** Developer can reach the dashboard immediately when running locally.
- **Simulated Sync:** UI transitions from "Pending" (⏳) to "Synced" (✅) after a realistic delay.
- **Production Safety:** All mock logic is physically removed (tree-shaken) from production builds.
- **Data Persistence:** Mocked sessions/tariffs persist in the local browser's IndexedDB.
- **Zero Cloud Impact:** No network requests reach the real Supabase backend in Test Mode.

## 3. Architecture

### 3.1 Network Interception (MSW)
- **Library:** [Mock Service Worker (MSW)](https://mswjs.io/)
- **Handlers:** 
    - `POST /auth/v1/*`: Return mock JWT and session.
    - `POST /rest/v1/*`: Intercept DB inserts/updates and return `201 Created` or `204 No Content`.
    - `GET /rest/v1/*`: Return empty arrays or mock seed data if requested.
- **Latency:** Randomized delay (300ms - 800ms) for all intercepted calls.

### 3.2 Integration Strategy
- **Conditional Loading:** MSW worker initialization is wrapped in `if (import.meta.env.DEV && import.meta.env.VITE_ENABLE_MOCKS)`.
- **Environment Variable:** `.env.local` will contain `VITE_ENABLE_MOCKS=true`.
- **Auth Hook Adaptation:** `useAuth` will detect `isMockMode` and return a static user object to bypass the login UI.

### 3.3 Data Flow
1. **User Action:** Submits a session in `SessionForm`.
2. **Local Save:** `sessionService` saves record to Dexie (IndexedDB) with `pending_sync: 1`.
3. **Sync Trigger:** `SyncEngine` detects the new record and makes a `fetch()` call to Supabase.
4. **Interception:** MSW intercepts the `fetch()` call, waits 500ms, and returns a successful response.
5. **Local Update:** `SyncEngine` receives the "success" and updates the local Dexie record to `pending_sync: 0`.
6. **UI Update:** `ChargingHistory` reflects the "Synced" state.

## 4. Components & Files
- `src/mocks/handlers.ts`: Defines the API responses.
- `src/mocks/browser.ts`: Configures the MSW worker.
- `public/mockServiceWorker.js`: The MSW service worker file (generated by CLI).
- `src/lib/mock-utils.ts`: Helper to detect mock mode via Vite env vars.

## 5. Testing & Validation
- **Manual Verification:** Add a session in Dev Mode and observe the sync indicator.
- **Build Verification:** Run `npm run build` and verify that the `msw` library and mock handlers are not present in the generated JavaScript chunks.

## 6. Future-Proofing
- **Seed Data:** This infrastructure will support a "Seed Data" utility in Phase 5 to populate the local DB with large datasets for analytics testing.
