# Design and Implementation Record Archive

The `specs/` and `plans/` directories preserve feature-design and implementation history. They are evidence of what was proposed or used during earlier work, not a current architecture guide or backlog. Paths, commands, data models, and “current” statements inside those dated records may describe the repository at that point in time.

Do not update historical records merely to match later refactors. Use these current sources instead:

- [`../../README.md`](../../README.md): project overview and documentation entry point
- [`../../CONTRIBUTING.md`](../../CONTRIBUTING.md): human engineering workflow
- [`../architecture.md`](../architecture.md): implemented current architecture
- [`../adr/`](../adr/): architectural decisions and supersession history
- [`../infrastructure-runbook.md`](../infrastructure-runbook.md): provisioning and deployment
